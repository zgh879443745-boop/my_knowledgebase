# 让Codex干活飞起 | Codex五大插件实测

## 视频摘要

Codex 最近火出圈了。作者做过一期插件实测视频播放量破百万，这次把天天在用的五个核心插件一次性讲清楚，涵盖动态页面制作、电脑操控、浏览器自动化、PPT 生成、编程开发五个维度。

## 核心观点

### 插件一：HyperFrame — 自然语言生成动态页面

用自然语言做出动态页面效果。动态视觉天然有审美优势，做演示比 PPT 动感，做视频能直接渲染出片。视频的开头和当前片段就是用 HyperFrame 做的。哪一帧不满意，告诉 Codex 改哪里、要什么风格，效果立等可见。

### 插件二：Computer Use — AI 的未来形态

最具想象力的插件，可能是 AI 的未来形态。之前只有 macOS 版，近日 Windows 版也上线。全能型——网页、文档、各种 APP，只要手动能点的，它都能自己点、自己看、自己理解、自己操作。目前还比较慢，但跑通后效率化只是时间问题。

### 插件三：Chrome — 浏览器自动化

5 月初上线，让 Codex 控制浏览器。能模拟操作电商、社交媒体、社区等网站，把信息扒出来。相比写 Playwright 爬虫脚本或用内置浏览器被登录卡住的旧方案，这个插件多标签运行、不影响正常使用。适合多网站信息汇总分析、企业内部系统自动化（填日报、报销等）。

### 插件四：Slides — 智能 PPT 生成

把一堆文件和素材扔进去，马上生成符合要求的 PPT 文档，还能套用公司模板。做完后自动检查文字和版式问题并自我修复。如果在提示词里加上"以美观的标准"这个指令，背景素材会通过 DALL·E 绘图模型生成，效果更好。

### 插件五：Coder — Vibe Coding 启蒙

走 Vibe Coding 这条路是迟早的事，随时开发自己的小工具令人上瘾。Coder 把一个顶级编程团队的协作方式拆解成了步骤：不会上来就写代码，而是先问问题、整理需求、理清思路，再告诉你该分几步做。多做几个项目，"先想清楚再动手"的思维就慢慢长进了脑子里，从会提需求变成会做事。

## 整体链路

这五个插件形成了完整的工作链路：
控制浏览器（Chrome）→ 做文档/PPT（Slides）→ 做动态内容（HyperFrame）→ 编程开发（Coder）→ 掌控整台电脑（Computer Use）

装好这五个插件，Codex 就能替你做日常重复工作，腾出时间和精力做更有价值的事。
